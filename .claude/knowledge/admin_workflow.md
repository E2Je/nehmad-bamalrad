# admin_workflow.md
_תהליכי עבודה של האדמין_

## כניסה לאדמין
קוד PIN: 06918 או 35321 → מאומת client-side + server-side ב-get-upload-token

## העלאת פרוטוקול (תהליך מלא)
1. בחירת קובץ → כותרת אוטומטית מהשם + הצעות תגיות
2. עריכת כותרת → הצעות תגיות מתעדכנות
3. בחירת קטגוריה
4. קבלת/דחיית תגיות + הוספה ידנית
5. לחיצת "הוסף פרוטוקול"
6. get-upload-token (245ms) → XHR לGitHub → register-protocol

## הצעת תגיות אוטומטית
- מופעלת על שם הקובץ + כותרת
- keyword map סטטית ב-suggestTags()
- מוגבלת ל-10 הצעות
- TODO: למידה מהמאגר הקיים (פרוטוקולים קיימים באותה קטגוריה)

## ניהול קטגוריות
- הוספה: emoji picker + שם → cat_${Date.now()} כ-ID
- מחיקה: confirm dialog → מסיר מהמאגר (פרוטוקולים נשארים, מוצגים כ"אחר")

## מחיקת פרוטוקול
- confirm dialog → מסיר מ-protocols.json
- הקובץ הפיזי נשאר ב-GitHub repo (ניתן למחוק ידנית אם נדרש)

## עריכת פרוטוקול
- ניתן לערוך: כותרת, קטגוריה, תגיות
- לא ניתן לערוך: קובץ (צריך למחוק ולהעלות מחדש)

## הגבלות העלאה
- תמונות: דחיסה אוטומטית ל-1920px/82% JPEG
- PDF/Word: עד 7MB client-side (server מקבל עד 10MB)
- כל סוג קובץ: עד 100MB (GitHub API limit)
